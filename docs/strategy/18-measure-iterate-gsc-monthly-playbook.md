# Monthly GSC Measure and Iterate Playbook

## Objective
Turn impressions into clicks with monthly micro-iterations on titles, descriptions, and FAQ snippets.

## Cadence
- Run once per month, same week each month.
- Compare last 28 days vs previous 28 days.

## Segment pages into groups
- Group A: Mortgage cluster
- Group B: Student loans cluster
- Group C: BMI/Calorie cluster
- Group D: Percentage cluster
- Group E: TR year-intent pages

## Core metrics (per page + per group)
- Impressions
- Clicks
- CTR
- Average position

## Prioritization rules
1. High impressions + low clicks (CTR under group median): first optimization target.
2. Rising impressions with flat CTR: snippet mismatch suspected.
3. Good CTR but low impressions: expand internal links and supporting pages.

## Micro-iteration framework (title/description)
- Version A (current) -> Version B (intent-first copy)
- Change one variable at a time:
  - title angle (calculator vs best vs how-to)
  - numeric cue (2026, examples, quick)
  - specificity cue (taxes+insurance, standard vs extended)
- Keep change log with date and page URL.

## FAQ iteration framework
- If a query variant appears repeatedly in GSC, add exact-match FAQ question.
- Keep answers short and direct (35-60 words).
- Re-test after 28 days.

## Monthly output template
1. Top 10 pages by impressions
2. Top 10 pages by CTR drop
3. Changes applied this month (title/description/FAQ/internal links)
4. Next month experiment queue (max 15 pages)

## Stop/continue criteria
- Continue variant if CTR improves by >= 15% with stable impressions.
- Revert or replace variant if CTR drops for 2 consecutive cycles.
